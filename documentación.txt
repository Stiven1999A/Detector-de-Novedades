Lo que se planea hacer es un modelo detector de anomalías siguiendo el siguiente esquema:
    1. Se realiza un modelo detector de datos atípicos.
    2. Se realiza un modelo detector de anomalías al ingresar datos nuevos.

A continuación, damos detalles de cómo ejecutar cada uno de los pasos:

    1. Modelo Detector de Anomalías:

    Usaremos dos métodos:
        i. LocalOutlierFactor.
        ii. Isolation Forest.
    
    Para realizar el modelo LocalOulierFactor debemos de usar el esquema ETL (Extraer, transformar y cargar). 
    Primero, extraemos los datos desde SQL Server, un histórico desde la fecha actual y convertimos la data en un dataframe.
    Segundo, para cada proceso único aplicamos el modelo de datos atípicos y determinamos cuáles son inliers y cuáles son outliers;
    considero que cada uno de estos datos, es decir tenemos para cada proceso su respectivo conjunto de datos normales o inliers. Estos
    datos deben de almacenarse de tal forma que solo sea necesario identificar el conjunto de datos normales una sola vez. 
    Una vez almacenados podrémos hacer uso de ellos muchas veces ahorrando tiempo de ejecución.
    Tercero, para cada proceso y conjunto de datos considerados inlier bajo el modelos debemos de realizar un
    modelo detector de anomalías usando One-Class SVM junto con los datos de nuevos que nos han llegado.

    El primer paso está realizado.
    
    El segundo paso debemos de planificarlo:
    debemos de guardar los nombres únicos de cada proceso (creo que desde la base de datos esto está hecho). Una
    vez hecho esto debemos de hacer un for para cada proceso, extraer los datos del histórico para ese proceso,
    aplicar el modelo con los respectivos parámetros, crear un dataframe con los valores considerados inliers por el modelo
    y almacenarlos de alguna manera de tal forma que pueda seguir consultandolos (esto se puede hacer creando una tabla en SQL Server)

    El tercer paso sería consultar la base de datos creada que contiene los dato típicos o inliers y aplicar el modelo 
    One-Class SVM con los datos nuevos para determinar si hay o no novedades. Cuales son esos proceso y dar métricas que respondan
    al por qué de su clasificación como outlier.
